Repositórios e commits

  • 1 - Crie uma conta: https://github.com/

  • 3 - Clique em Clonar o repositório

  • 5 - Num terminal digite o comando: $ git clone endereco_copiado
  • 6 - Dentro da pasta desejada:
  • $ git config user.email "email@email.com"
  • $ git config user.name "Seu Nome"
  • $ git add .
  • $ git commit -m "Commit ajustando pastas no repositório"
  • $ git push

Heroku - Servidor Cloud

Estes são alguns atalhos no VSCode para aumentar a produtividade.

    No Linux Ubuntu 16+

  • 01 - Instalar: $ sudo snap install --classic heroku
  • 02 - Criar conta no Heroku
  • 03 - Criar um App no site (copie a url para usar no passo 5)
  • 04 - Terminal: $ heroku login -i
  • 05 - Clonar App: $ git:clone -a https://git.heroku.com/myapp.git
  • 06 - Criando vínculo: $ heroku git:remote -a nomeDoApp
  • 07 - Edite localmente o app (VSCode)

    Preparando para subir a aplicação:

  • 08 - Verificar modificações: $ git add .
  • 09 - Commit: $ git commit -m "Minhas alterações"
  • 10 - $ git push heroku
  • Site do Heroku: https://www.heroku.com/

ReactJS - Biblioteca do JavaScript

O ReactJS não é uma linguagem de programação e nem tampouco um Framework. Trata-se de uma biblioteca de JavaScript desenvolvida pelos engenheiros do Facebook para facilitar suas rotinas internas no desenvolvimento de interfaces de usuários em algumas aplicações mas que acabou sendo liberada como open source.

    Instalação

  • Instalar o npm (Instalador de pacotes JavaScript): $ sudo apt-get install npm
  • Crie uma pasta onde irá instalar o projeto: $ mkdir meuprojeto
  • Entre na pasta: $ cd meu_projeto
  • Dentro da pasta: $ npm init
  • Crie a pasta 'public': $ mkdir public
  • Crie a pasta 'src': $ mkdir src
  • Crie um arquivo chamado 'index.html' dentro de 'public'
  • Crie um arquivo chamado 'index.js' dentro de 'src'
  • Crie um arquivo chamado 'styles.css' dentro de 'src'
  • Coloque alguma formatação css. Ex: body { color: blue; }
  • Na pasta principal (meu_projeto): $ npm install --save react@16.8.6 react-dom@16.8.6 react-scripts@3.0.1
  • Editar o arquivo package.json com as configurações necessárias.
  • Após instalado levantar o servidor: $ npm start

Importante:
O nome do seu projeto não pode ser o nome do diretório. Caso contr ário um erro será gerado como abaixo:
$ $ npm ERR! Refusing to install react as a dependency of itself
Se precisar alterar o nome do projeto edite o arquivo package.json gerado após o comando anterior 'npm init'. Troque o "name" para qualquer nome diferente do diretório onde o projeto está sendo criado.

Detalhe: (não é uma boa prática mas pode ser útil)
Após ter rodado todos os comandos e efetuado todas as configurações para que a aplicação possa subir, pode-se copiar todo o conteúdo do diretório atual para um novo diretório e efetuar as apropriados customizações nos arquivos package.json, index.html, index.js e styles.css e a nova aplicação estará pronta para subir. Caso nenhuma alteração seja efetuada a aplicação que subirá no servidor será um clone da anterior.
